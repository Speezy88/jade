# Phase 1 Spec — Morning Briefing
Version: 1.0 | Status: Approved
Last updated: 2026-03-05

---

## 1. What and Why

Phase 1 ships a 7am morning briefing that gives Spencer a situational read on his day
before he leaves for school. It replaces the passive, reactive start to the morning with
a structured, data-grounded summary of what actually matters today.

This is the foundation everything else builds on. SOUL.md and ACTIVE_GOALS.md injection
happens here first. If this doesn't run cleanly every morning, nothing downstream is reliable.

**Hard dependencies (Phase 1 cannot start without these):**

- .env contains all 3 keys: ANTHROPIC_API_KEY, SCHOOLOGY_ICS_URL, OPENWEATHERMAP_API_KEY

## 3. Output — Briefing Format

Delivered in this exact order:

```
[GREETING + WAKEUP]
Good morning Spencer. [Day-appropriate greeting based on schedule density.]

[DATE + WEATHER]
Today is [Weekday, Month Day]. [Weather: current conditions, high/low for Seattle.]

[SCHOOLOGY — ASSIGNMENTS DUE]
Assignments due today or tomorrow:
• [assignment] — [class] — due [date]
• ...
[If fetch failed: "Schoology unavailable — check manually."]
[If no assignments: "No assignments due in the next 48 hours."]

[GOOGLE CALENDAR — TODAY'S EVENTS]
Today's schedule:
• [time] — [event]
• ...
[If zero events: "No calendar events today."]

[INSPIRATIONAL QUOTE]
[Single quote generated by Haiku, grounded in Spencer's ACTIVE_GOALS.md.
Not generic. Tied to something Spencer is actually working toward.]
```

**Delivery:** Terminal output + macOS notification (notification shows first 2 lines only).
**On any fetch error:** Always notify. Never fail silently.
**API call max tokens:** 150 (quote only — forces concision).

## 4. Ideal State Criteria (ISC)

All must pass before Phase 1 is marked complete:

- [ ] `jade_briefing.py` runs to completion with zero unhandled exceptions
- [ ] Output contains all 5 sections in correct order
- [ ] Google Calendar section shows live data (not hardcoded), verified against actual calendar
- [ ] "No calendar events today" appears correctly on a day with no events (manual test or mock)
- [ ] Schoology section shows live data when feed is reachable
- [ ] "Schoology unavailable" message appears when feed is unreachable (kill the URL, rerun)
- [ ] Weather shows correct conditions for Seattle (cross-check with phone)
- [ ] Quote is goal-specific — references something from ACTIVE_GOALS.md, not generic
- [ ] macOS notification fires with first 2 lines of briefing
- [ ] launchd fires `jade_briefing.py` at 7am without manual trigger (verify next morning)
- [ ] All errors write to `logs/briefing_error.log`
- [ ] Script exits 0 on success, non-zero on fatal error
- [ ] Total API cost per run ≤ $0.01 (Haiku, 150 tokens max)

## 6. Implementation Constraints

From TOOLS.md — must read before building:

**Google Calendar:**
- OAuth 2.0 — not API key. credentials.json → ~/.config/jade/, never in repo
- Google returns UTC — convert to America/Los_Angeles before display
- All-day events have `date` not `dateTime` — handle both or crash
- First run requires browser auth — pre-auth manually before loading launchd plist
- Token auto-refreshes if refresh token exists in token.json

**Schoology:**
- Full ICS export includes past events — filter `DTSTART >= today`
- Some DTSTART is date, some datetime — handle both
- Cache to memory/cache/schoology.json, 6h TTL
- Feed lags ~6h after teacher posts — this is expected, not a bug

**OpenWeatherMap:**
- Free tier: 60 calls/min, 1000 calls/day — far more than needed
- Endpoint: `api.openweathermap.org/data/2.5/weather`
- Seattle coords: lat=47.6062, lon=-122.3321
- Units: imperial (Fahrenheit)
- Add OPENWEATHERMAP_API_KEY to .env

**launchd:**
- Does NOT inherit shell PATH — use absolute paths everywhere in plist
- .env not auto-loaded — use `load_dotenv("/Users/spencerhatch/Jade/.env")` with absolute path
- Pre-auth Google Calendar before loading plist — launchd cannot do browser auth
- Test manually before loading: `python3 jade_briefing.py`
- Check logs/briefing_error.log before assuming job ran

**API model string (exact):** `claude-haiku-4-5-20251001`

**Cost ceiling:** ≤$3/month total for all briefings (~$0.01/run × 30 days = $0.30 actual)

## 7. Gray Areas — Resolved

| Question | Decision |
|----------|----------|
| Zero calendar events | Show "No calendar events today." — do not skip section |
| Schoology fetch fails / stale >6h | Show "Schoology unavailable — check manually." — do not skip section |
| Error handling | Always send macOS notification on error — never fail silently |
| Output destination | Terminal + macOS notification (notification = first 2 lines) |
| Schoology in Phase 1? | Yes — include. Weather + Schoology + Calendar + Quote all ship in Phase 1 |
| Nightly briefing | Deferred to Phase 1.5 |
| Quote source | Claude Haiku generates from ACTIVE_GOALS.md content |

## 9. Build Sequence

In this order — do not reorder:

1. Write `ACTIVE_GOALS.md` (no code, 20 min) ← BLOCKED until this exists
2. Write `jade_prompts.py` — `build_system_prompt()` only, no briefing logic yet
3. Write `integrations/weather.py` — simplest integration, good warm-up
4. Set up Google Calendar OAuth — follow TOOLS.md exactly, pre-auth in browser
5. Write `integrations/gcal.py` — read-only fetch
6. Write `integrations/schoology.py` — ICS fetch + cache
7. Write `jade_briefing.py` — wire all integrations + build_system_prompt()
8. Test manually: `python3 jade_briefing.py` — verify all 5 sections
9. Verify all ISC criteria (Section 4)
10. Load launchd plist — verify fires next morning
11. Run `/update-docs` → `/retro` → close phase

## 10. Phase 1.5 Preview (do not build now)

Nightly briefing — `jade_nightly.py` — interactive, conversational:
1. Jade greets, asks how highest-priority events went
2. Jade logs data from responses (manual_log.csv, notes)
3. Jade asks about struggles and holdbacks
4. Jade helps plan tomorrow (proposes time blocks)
5. Optional: brainstorming session on active goals

This runs at ~9pm via a second launchd plist.
Requires interactive terminal session — not fire-and-forget like the morning briefing.
